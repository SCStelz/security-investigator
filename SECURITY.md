# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest on `main` | Yes |
| Older commits | No |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Open a GitHub Issue** with the label `security` describing the vulnerability at a high level. Do not include exploit details in public issues.
2. If the vulnerability is sensitive (e.g., credential exposure, remote code execution), request a private channel in the issue and a maintainer will follow up.
3. You can expect an initial response within 72 hours.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Scope

This policy covers the code and configuration in this repository. Third-party dependencies should be reported to their respective maintainers.

## Disclosure Policy

We follow coordinated disclosure. Please allow a reasonable timeframe for a fix before public disclosure.
